# diplom

1. Перечень автоматизируемых сценариев

Успешная оплата (валидная карта).
Отказ в оплате (невалидная карта).
Отказ в оплате (карта с недостаточным балансом).
Успешное одобрение кредита (валидная карта).
Отказ в выдаче кредита (невалидная карта).
Проверка ввода некорректных данных (неверный формат, пустые поля).
Проверка подсказок при ошибках ввода.

2. Перечень используемых инструментов
Java 
Selenium  - для автотестов UI
JUnit 5 - запуск тестов
Gradle 
Docker - запуск контейнеров с БД и приложениями
Allure Report - генерация отчетов

3. Возможные риски
Изменение структуры веб-страницы → Локаторы могут сломаться.
Проблемы с интеграцией → Нестабильная работа платежного шлюза/кредитного сервиса.
Разные данные в тестовой и продакшн-среде → Ошибки при проверке БД.

4. Интервальная оценка с учётом рисков
Настройка окружения и проекта- 4-6 ч
Написание тестов - 15-20 ч
Исправление замечаний - 8-16ч
Подготовка отчетов - 8ч

5. План сдачи работ
Готовность автотестов → 1 неделя.
Результаты прогона → 2–3 дня после завершения тестов.
Отчёт по автоматизации → 1 день после анализа результатов.
